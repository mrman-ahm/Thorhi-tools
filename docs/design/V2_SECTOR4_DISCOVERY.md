# THROHI V2 — Sector 4 Discovery System

## Typography calibration

The user approved the bold editorial direction but requested slightly smaller display typography in the most oversized scenes.

The following groups are reduced by approximately 8–12% depending on viewport:

- hero declaration
- major section introductions
- division names
- instrument-function words
- macro and history headings
- catalogue, inquiry, proof, and contact declarations
- large footer treatment where its class is available

Body copy, technical metadata, product codes, form labels, navigation, and controls remain unchanged.

## Division discovery

Desktop uses two coordinated columns:

- a sticky active-stage showing the current division index, generic instrument geometry, description, and function words
- four large direct catalogue links that naturally activate while scrolling, focusing, or pointing

The active atmosphere changes among green, blue, amber, and coral while the overall THROHI dark system remains stable.

Mobile removes the sticky stage and presents the four divisions as intentional stacked catalogue entrances.

No information is exclusive to the active-stage animation. Each link includes its own name, description, functions, and route.

## Functional-language passage

The homepage now connects instrument functions directly to search routes:

- Cut
- Hold
- Retract
- Suture
- Examine
- Extract

The words are large but calibrated below the earlier oversized passage. Movement occurs only on hover or keyboard focus.

## Family discovery

Desktop uses normal vertical page scrolling to progress a horizontal family track.

- the wheel and scrollbar are never intercepted
- no forced snapping is used
- all panels are ordinary links
- a progress line shows passage position
- scroll calculations use `requestAnimationFrame`
- the track uses one transform write per animation frame

Mobile becomes a single-column vertical archive.

Reduced motion becomes a static two-column grid on larger screens and one column on narrow screens.

## Protected behavior

Sector 4 does not change:

- catalogue data
- search ranking
- inquiry storage or migration
- quantity, notes, remove, or undo
- inquiry API behavior
- product and family routes

## Explicit exclusions

- macro inspection interaction
- scissors-evolution sequence
- image flipping
- Easter eggs
- smooth scrolling
- final photography
- 3D assets or rendering
